MyFlix (Angular)

Angular | Angular Material

Overview

This is the front-end of a previous project that I built. It is a RESTful API that I built with Node.js and React. This project can be viewed here. I built this front-end with Angular and Angular Material. It includes all the same endpoints, allowing for users to create accounts, view movies and their info (genre, director, actors, synopsis) as well as add movies to their favorites list. Users can also edit some of their profile information. This app primarily uses modals to display data.

The Journey

Project Management

I first began my journey by learning about collaboration with designers and project management techniques and common materials. This was primarily focused on the very common Agile methods, Scrum and Kanban. After learning about the different types of meetings that take place with Scrum, such as Sprint Planning, Stand-Up, and Iteration ration Review, I began learning about and working on Kanban boards. Upon learning about this type of framework, I built my own for this project. My board consisted of 3 columns, "To-Do, "In Progress", and "Done".

TypesScript Angular Material

The project began with an introduction to the installation process of Angular 13, as well as using the CLI. I learned about how the modules worked in Angular, as well as data binding and templates. I found it interesting that components could be simulated for testing. Finally, I practiced TypesScript. The actual project began with creating my endpoints in the "fetch-api-data.service.ts" file, then generating components and adding dialog boxes and cards via Angular Material. I also created a navbar and sidenav for navigation, as well as functions for changing the "favorite" icons in the movie cards so users have the option of selecting movies to add to their favorites list. The final step was documenting my code via TypeDoc. Although Angular has a steep learning curve and definitely proposed many challenges, I got help and was able to finish the project, and I am very happy to have been exposed to this framework as well as TypesScript.
